# 专家级奖励系统设计 v6.0

## 🎓 核心设计理念

### 1. 奖励尺度一致性原则
**问题**：比赛分数 ≠ 强化学习奖励
- 比赛规则：笑脸4分、红包2分、终点+时间
- RL需求：根据训练效果动态调整权重

**解决**：重新设计总奖励预算，确保里程碑奖励占主导

### 2. 总奖励预算分配（单episode）

```
理想总奖励 = 115分
├─ 过程奖励（密集，每步给予）：15分 (13%)
│  ├─ 前进奖励：~10分
│  └─ 磁吸引导：~5分
│
├─ 里程碑奖励（稀疏，触发给予）：75分 (65%)
│  ├─ 笑脸收集：3 × 10 = 30分
│  ├─ 红包收集：3 × 5 = 15分
│  ├─ 完整到达：50分
│  └─ 庆祝动作：5分
│
└─ 失败策略（直冲终点）：18分
   ├─ 前进：~10分
   └─ 跳过到达：8分

差距比例：115 / 18 = 6.4倍 ✅
```

## 📊 详细奖励配置

### 一、过程奖励（密集，持续给予）

#### 1.1 前进奖励
```python
forward_reward = clip(Vy × 0.005, -0.01, 0.01)
```
**计算预期**：
- Episode步数：2000步
- 平均前进速度：Vy = 0.5 m/s
- 每步奖励：0.5 × 0.005 = 0.0025
- Episode总计：2000 × 0.0025 = **5分**
- 最优前进（Vy=2m/s）：2000 × 0.01 = **20分**

**设计意图**：
- 权重0.005，远小于地标奖励（10分）
- 确保机器人有前进动力，但不会淹没里程碑奖励
- 梯度足够明显，PPO能学到"往前走有奖励"

#### 1.2 磁吸引力
```python
for 未收集地标:
    if dist < 3.0:
        strength = (3.0 - dist) / 3.0
        attraction = strength × clip(Vx×direction × 0.002, 0, 0.005)
```
**计算预期**：
- 靠近地标时触发：距离1.5m时strength=0.5
- 横向移动Vx=0.5m/s时：0.5 × 0.001 = 0.0005/步
- 收集期间约1000步：1000 × 0.0005 = **0.5分**
- 6个地标总计：~**3分**

**设计意图**：
- 权重0.002，仅作为引导信号
- 不干扰主要前进奖励
- 距离越近吸引力越强，自然涌现"靠近就横向移动"

#### 1.3 边界警告
```python
if dist_to_boundary < 1.0:
    penalty = -0.5 × (1.0 - distance)  # 每步最大-0.5
```
**设计意图**：
- 轻度持续惩罚，不是终止
- 引导机器人远离边界，但不过分限制
- 最大每步-0.5，100步累计-50分（严重但不致命）

### 二、里程碑奖励（稀疏，触发给予）

#### 2.1 笑脸收集
```python
smile_reward = 10.0分/个  (不是比赛的4分！)
总计：3 × 10 = 30分
```
**触发条件**：
- 进入矩形区域：|X - lx| < 0.5 且 |Y - ly| < 0.5
- 首次触发，重复进入不加分

**设计意图**：
- 10分远大于前进过程奖励（~10分total）
- PPO会发现"收集笑脸比直冲终点更有价值"
- 稀疏但足够吸引人

#### 2.2 红包收集
```python
package_reward = 5.0分/个  (不是比赛的2分！)
总计：3 × 5 = 15分
```
**触发条件**：
- 进入矩形区域：|X - lx| < 0.3 且 |Y - ly| < 0.3
- 红包区域更小（0.6m vs 1.0m），难度稍高

**设计意图**：
- 5分低于笑脸（10分），符合任务优先级
- 总计15分仍然重要，不能忽略

#### 2.3 终点到达（条件化）
```python
完整收集到达：50分  (6个地标全收集)
跳过收集到达：8分   (差距6.25倍！)
```
**触发条件**：
```python
if all_collected:
    goal_reward = 50分
else:
    goal_reward = 8分  # 严厉惩罚
```

**设计意图**：
- 50分是最大单项奖励，强激励
- 8分跳过惩罚远低于"前进+收集"组合
- 差距6.25倍，PPO会被迫学习收集路径

#### 2.4 庆祝动作
```python
celebration_reward = 5.0分
```
**触发条件**：
- 到达终点后
- Z速度过零3次（连续蹦3次）

**设计意图**：
- 额外奖励，增加趣味性
- 5分不影响主要策略

### 三、惩罚系统（分级设计）

#### 3.1 严重违规（立即终止）
```python
摔倒（tilt > 60°）：-50分
出界（X/Y边界）：-50分（隐含在终止中）
物理发散（NaN）：-50分（隐含在终止中）
```
**设计意图**：
- -50分足够严重，但不至于崩溃训练
- 相比最优策略115分，摔倒损失惨重
- PPO会学到"避免摔倒"的强信号

#### 3.2 轻度违规（持续惩罚）
```python
边界警告（1m内）：-0.5/步 × (1-distance)
卡住不动：终止episode（无额外惩罚）
```

#### 3.3 稳定性惩罚（微小持续）
```python
角速度惩罚：-0.005 × (ωx² + ωy²)
```
**设计意图**：
- 极小权重，避免过度抖动
- 不影响主要奖励信号

## 🎯 训练预期行为

### 阶段1：学会站立+前进（Epoch 0-1000）
```
回报：-10 → 15分
行为：
- 学会不摔倒（避免-50）
- 发现前进有奖励（+0.005/步）
- 开始往前走，可能冲出边界
```

### 阶段2：探索发现地标（Epoch 1000-5000）
```
回报：15 → 40分
行为：
- 某次偶然进入笑脸区域 → +10分！
- PPO记录高价值经验
- 开始尝试横向移动靠近地标
- Advantage函数：40分 vs 15分 = 巨大差异
```

### 阶段3：学会收集策略（Epoch 5000-15000）
```
回报：40 → 90分
行为：
- 策略梯度强化"前进+磁吸+收集"循环
- 学会横向扫描（左→中→右）
- 发现完整收集能拿50分终点奖励
- 最终策略：前进→笑脸层→横扫→红包层→横扫→终点
```

### 阶段4：精修最优路径（Epoch 15000-30000）
```
回报：90 → 115分
行为：
- 优化横向扫描轨迹效率
- 学会在2026平台庆祝（+5分）
- 减少无效探索，提高成功率
- 稳定在110-115分
```
